백엔드 디렉토리 구조 및 역할

관심사 분리(Separation of Concerns) 원칙에 따라 분산 설계된 백엔드 핵심 구조입니다.

📁 backend/app
├── 📁 api/ # [컨트롤러] 클라이언트의 HTTP 요청 진입점 및 권한 가드
│ ├── 📄 deps.py # 공통 의존성 주입 (현재 로그인 유저 확인 등)
│ └── 📁 v1/endpoints/ # 기능별 라우터 분할 (auth, posts, events)
├── 📁 core/ # [공통설정] 데이터베이스 연결 설정, 보안 유틸리티
│ ├── 📄 config.py # 환경 변수 및 JWT 비밀키 로드
│ ├── 📄 database.py # SQLAlchemy 연결 객체 생성
│ └── 📄 security.py # 패스워드 암호화 해싱 및 JWT 생성
├── 📁 crud/ # [데이터 질의] DB 테이블 생성/조회/수정/삭제 실제 SQL 로직 수행
│ ├── 📄 base.py # 공통 CRUD 부모 추상 클래스
│ └── 📄 crud_user.py / crud_post.py ... # 도메인별 쿼리 분리
├── 📁 models/ # [DB 테이블 매핑] 데이터베이스 실제 테이블 형태와 매핑되는 파이썬 클래스
├── 📁 schemas/ # [입출력 검증] 외부 요청의 데이터 유효성을 검사하는 Pydantic 스키마
└── 📄 main.py # 🚀 백엔드 기동 엔트리포인트 (기본 어드민 계정 자동 생성)

백엔드 핵심 기술 개념

FastAPI 백엔드를 구성하는 핵심 3대 기술 컴포넌트의 역할입니다.

FastAPI Web Framework

현대적이고 매우 빠른 파이썬 웹 프레임워크입니다. 코드 작성과 동시에 스웨거 문서(/docs)를 자동 빌드해 주는 막강한 기능을 제공합니다.

SQLAlchemy ORM (Object-Relational Mapping)

데이터베이스의 테이블을 파이썬 클래스처럼 다룰 수 있게 해주는 도구입니다. 복잡한 SQL문을 한 줄의 파이썬 객체 메서드로 실행하게 돕습니다.

Pydantic Data Validation

외부에서 전송된 JSON 데이터가 누락되었거나 형식이 맞지 않는 경우, 백엔드 진입 전 자동으로 유효성을 검증하고 422 에러를 반환해 서버를 보호합니다.

SQLite (app.db) File Database

별도의 DB 서버 설치 없이 단일 파일 단위로 돌아가는 경량 파일 데이터베이스입니다. 로컬 개발 시 매우 빠르고 간편하게 사용 가능합니다.

API 요청 처리 흐름 (Data Flow)

클라이언트에서 요청이 도달했을 때 백엔드 내부에서 데이터가 흐르는 기본 경로입니다.

1

HTTP Request 진입 (API Layer)

브라우저가 요청을 보냅니다. api/v1/endpoints/의 라우터가 Pydantic 스키마(schemas/)를 사용해 데이터 유효성을 검사합니다.

2

의존성 및 인증 가드 작동 (Security Layer)

api/deps.py가 동작하여 헤더에 포함된 JWT 토큰을 해석하고 유효한 사용자인지, 관리자인지 권한을 검증합니다.

3

비즈니스 쿼리 요청 (CRUD Layer)

검증을 마친 요청은 crud/의 매핑 인스턴스로 넘겨집니다. 이곳에서 실제 필요한 데이터베이스 조회 및 조작 함수를 구동합니다.

4

데이터베이스 매핑 및 반영 (ORM Layer)

SQLAlchemy 모델(models/) 객체를 조작하여 SQLite(app.db)에 물리적인 데이터를 기록하고 영구 보관합니다.
