프론트엔드 디렉토리 구조 및 역할

프로젝트의 기능(도메인)단위 격리 설계에 기반한 핵심 폴더들의 레이아웃입니다.

📁 frontend/src
├── 📁 api/ # 백엔드 서버(FastAPI)와 통신하기 위한 Axios 기본 설정
│ └── 📄 client.ts
├── 📁 features/ # 핵심 비즈니스 로직(기능) 컴포넌트 모음
│ ├── 📁 auth/ # 🔐 회원가입/로그인/내 정보 수정 및 회원 목록 제어
│ │ ├── 📁 components/ # 화면 컴포넌트 (.tsx)
│ │ └── 📁 hooks/ # 인증 및 가입 API 상태 관리 훅 (.ts)
│ ├── 📁 posts/ # 📝 자유게시판 글쓰기/목록/상세 및 댓글 기능
│ │ ├── 📁 components/
│ │ └── 📁 hooks/
│ └── 📁 events/ # 📅 일정 공유 달력, 일정 등록 팝업
│ ├── 📁 components/
│ └── 📁 hooks/
└── 📄 App.tsx # 🏠 메인 레이아웃 및 탭 화면 분기 컨트롤

웹 확장자별 기능 및 역할

프론트엔드 소스코드 구성에 활용된 확장자 파일들의 쓰임새를 쉽게 비교 정리했습니다.

.tsx

React 컴포넌트 파일

HTML(화면 뼈대)과 TypeScript(동작 로직)를 결합하여 화면에 그려지는 조각(컴포넌트)을 짤 때 사용합니다.

.ts

순수 TypeScript 파일

화면을 그리는 코드 없이, 백엔드 서버와의 API 요청/응답 처리나 데이터 가공과 같은 핵심 로직만을 정의할 때 사용합니다.

.css

스타일시트 디자인 파일

웹 화면의 색상, 정렬, 폰트, 호버 효과 및 그림자 등 시각적이고 감성적인 스타일을 적용할 때 활용합니다.

.json

설정 데이터 파일

설치할 라이브러리 목록 명세(package.json)나 TypeScript 컴파일러 컴파일 규칙 등 메타 설정값을 저장하는 텍스트 파일입니다.

.html

기본 마크업 뼈대

브라우저가 처음 해석하는 웹 페이지입니다. React는 이 파일 내부에 실시간으로 컴포넌트들을 끼워넣어 화면을 갱신합니다.

.svg

벡터 그래픽 파일

화면 크기를 아주 크게 키우거나 작게 줄여도 화질이 절대 깨지지 않는 무손실 고품질 벡터 아이콘용 이미지 규격입니다.

💡 쉽게 말해, .tsx로 화면 모양을 조각하고, .ts로 연동 기능을 연결하며, .css로 예쁜 테마 옷을 입힌 뒤 .html을 통해 브라우저 화면에 띄우는 것이 프론트엔드의 동작 원리입니다!
