# 📜 Project Development History

이 문서는 "카페 모임" 프로젝트 개발 과정에서 발생한 모든 사용자 요청(프롬프트)과 개발 단계별 작업 진행 이력을 순차적으로 기록한 개발 히스토리 파일입니다.

---

## 📅 1. 기획 및 아키텍처 설계 단계

### [요청 1] 로그인 보안 방식 문의
* **내용:** 로그인 시 보안을 어떤 방식으로 처리할 것인지에 대한 설계안 요청
* **반응:** 단방향 해싱(bcrypt)과 JWT 토큰 기반 인증 프로세스 설명 완료 및 동의.

### [요청 2] DB 전환 용이성 검토
* **내용:** "SQLite를 추후 MySQL로 바꾸는 과정이 어렵지 않은가?"에 대한 기술적 질의.
* **반응:** SQLAlchemy ORM을 사용하여 DB 드라이버 환경설정 변수만 수정하면 코드 변경 없이 포팅 가능한 설계 방향 제시 및 확인.

### [요청 3] 기능별 디렉토리 분할 리팩토링 지시
* **내용:** "시작하기 전에 소스를 보니 너무 코드가 한곳에 모여 있는것 같아 기능별로 디렉토리를 만들어 openapi/backend/frontend로 분리해줘. 단, SQL은 한곳에서 계속 관리해줘."
* **반응:** 
  - 백엔드(FastAPI)의 모델, 스키마, CRUD, API 경로를 도메인(Auth, Post, Comment, Event)별 개별 파일로 분리.
  - OpenAPI 규격도 각 도메인별 YAML 파일로 분산 설계하고 `main.yaml`에서 조합하도록 쪼갬.
  - 데이터베이스 SQL은 기존대로 `/backend/sql` 기조 하에 일괄 관리.

## 📅 2. 핵심 기능 개발 및 로컬 1차 검증 단계

### [요청 4] 회원가입, 자유게시판 및 일정 기능 기본 구현
* **내용:** 분할된 백엔드/프론트엔드 구조 하에 사용자 가입, 로그인, 게시판 및 캘린더 기본 기능 개발 착수 및 진행.
* **반응:**
  - 백엔드 데이터베이스 SQLAlchemy 및 Pydantic 뼈대 수립.
  - 가상환경 설정 및 DB 자동 테이블 생성 완료.
  - 프론트엔드 React + Vite 보일러플레이트 구성 후 API 연동용 커스텀 훅 작성 및 컴파일 테스트.

### [요청 5] 로컬 실행 및 버그 수정
* **내용:** "로컬에서 실행해줘"
* **반응:** 
  - 백엔드 uvicorn 개발 서버 및 프론트엔드 Vite 개발 서버 구동.
  - 구동 도중 파이썬 passlib의 bcrypt bug(패딩 길이 체크 에러) 트러블슈팅을 발견하여, passlib 대신 내장 `bcrypt` 모듈을 직접 호출하는 방식으로 즉시 소스 수정 및 기동 성공.

## 📅 3. 어드민 기능 및 회원 정보 수정 추가 단계

### [요청 6] 관리자용 회원 관리 기능 추가 지시
* **내용:** "관리자의 회원 관리 기능이 빠진것 같은데"
* **반응:** 관리자(`admin`) 로그인 시 가입 멤버 목록을 확인하고 강제 탈퇴(삭제)시킬 수 있는 `GET /auth/users` 및 `DELETE /auth/users/{user_id}` API 및 [MemberList.tsx] 화면 구현.

### [요청 7] 일반 회원의 정보 수정 기능 추가 지시
* **내용:** "그리고 회원의 회원 정보 수정 기능도 넣어줘"
* **반응:** 로그인한 사용자가 자신의 닉네임과 패스워드를 바꿀 수 있는 `PUT /auth/me` API 및 [ProfileEdit.tsx] 컴포넌트 추가 완료.

## 📅 4. 캘린더 일정 다각화 및 UI 튜닝 단계

### [요청 8] 종일 및 다중일(기간) 일정 지원 지시
* **내용:** "일정관리 기능에 종일 일정 등록 기능과 여러날에 걸친 일정도 등록 할수 있는 형태로 바꾸어줘"
* **반응:**
  - Event DB 모델 및 스키마에 `is_all_day` 컬럼 추가 및 DB 테이블 재생성.
  - 등록 모달에 시작일/종료일 날짜 선택 UI 및 종일 여부 체크박스 추가.
  - 시작 날짜~종료 날짜 범위에 걸쳐 달력 칸에 일정이 연속 표시되도록 렌더링 로직 고도화.

### [요청 9] 시간 설정 UI 교체 및 종일 일정 스타일링 조정
* **내용:** "일정 등록시 시간 설정 드롭다운 메뉴 표시가 이상해 오전/오후로 하지 말고 0~23시 선택으로 바꾸어 주고 종일일정 표시할때 \"[종일]\" 용어 빼고 색으로만 구분해줘"
* **반응:**
  - 기본 시간 인풋 창을 제거하고 직관적인 `00시~23시` 및 `00분~59분` HTML select 태그 기반 드롭다운 메뉴 적용.
  - 달력 일정 칩에서 `[종일]` 마커 텍스트를 제거하고, 구글/네이버 캘린더 스타일의 **꽉 찬 솔리드 색상(어드민: 빨강, 멤버: 파랑)과 흰색 텍스트** 스타일링 적용.

### [요청 10] 로컬 검증 로그 데이터 초기화 원인 질의
* **내용:** "지금 테스트를 하는데 전에 넣었던 데이터를이 계속 없어지는데 이유는 뭐니?"
* **반응:** 종일 일정 추가에 따른 DB 테이블의 컬럼 구조(`is_all_day`) 스키마 변경 시, 로컬 SQLite DB 파일(`app.db`)을 삭제하고 새롭게 재생성하면서 데이터가 리셋되었음을 답변 및 안내.
